# Guia Operacional: Avaliações e PDI

Este guia resume o uso do ciclo de Avaliação + PDI no Sistema Administrativo Hotel Galli.

O objetivo é ajudar RH e gestores a acompanhar experiência, desempenho e desenvolvimento de colaboradores de forma simples, prática e rastreável.

## 1. Visão geral

O ciclo operacional é:

```text
Modelo de avaliação
-> Avaliação do colaborador
-> Notas e comentários
-> Devolutiva
-> Ciência do colaborador
-> Conclusão
-> Relatório
-> PDI, quando necessário
```

A avaliação apoia a decisão humana do RH e do gestor. Ela não gera promoção, punição ou desligamento automático.

## 2. Quando usar avaliação

Use avaliação para registrar:

- acompanhamento de experiência;
- avaliação periódica;
- desempenho operacional por setor;
- pontos fortes;
- pontos que precisam de acompanhamento;
- devolutiva feita com o colaborador.

Não use avaliação como advertência.

## 3. Quando usar PDI

Use PDI quando a avaliação mostrar necessidade de acompanhamento prático.

O PDI serve para combinar ações, prazos e acompanhamento. Ele não é punição, advertência ou promessa automática de promoção.

## 4. Papéis envolvidos

- RH: configura modelos, acompanha relatórios e orienta o processo.
- Gestor: aplica avaliação, registra devolutiva e acompanha PDI.
- Colaborador: toma ciência da avaliação e participa das ações combinadas.

Ciência significa que o colaborador tomou conhecimento. Não significa concordância obrigatória.

## 5. Como configurar modelos

Acesse:

```text
RH -> Modelos de avaliação
```

Nesta tela o RH pode:

- criar modelo;
- usar modelos padrão Hotel Galli;
- configurar seções;
- configurar critérios;
- marcar critério crítico;
- definir comentário obrigatório para nota baixa;
- ativar ou inativar modelo.

Somente modelos ativos aparecem para aplicação em colaboradores.

## 6. Como aplicar avaliação

Acesse:

```text
RH -> Colaboradores -> abrir colaborador -> aba Avaliações
```

Fluxo recomendado:

1. Clique em `Nova avaliação`.
2. Escolha um modelo ativo.
3. Informe período e data.
4. Preencha as notas dos critérios.
5. Preencha comentários obrigatórios quando houver nota baixa crítica.
6. Salve as notas.

Critérios críticos aparecem destacados. Nota baixa em critério crítico precisa de comentário.

## 7. Como registrar devolutiva

Depois de preencher a avaliação:

1. Marque a avaliação como pronta para devolutiva.
2. Converse com o colaborador.
3. Preencha data da devolutiva.
4. Registre resumo, pontos fortes e pontos a desenvolver.
5. Clique em `Registrar devolutiva`.

A devolutiva deve ser objetiva e ligada ao trabalho observado.

## 8. Como registrar ciência

Depois da devolutiva:

1. Registre comentário do colaborador, se houver.
2. Clique em `Registrar ciência`.

Ciência não significa concordância. Significa que o colaborador foi informado sobre a avaliação.

## 9. Como concluir avaliação

Depois da ciência:

1. Revise notas, comentários e devolutiva.
2. Clique em `Concluir avaliação`.

Após concluída, a avaliação fica bloqueada para edição operacional de notas e critérios.

## 10. Como criar PDI

Acesse:

```text
RH -> Colaboradores -> abrir colaborador -> aba Desenvolvimento
```

Ou, se estiver na avaliação e houver ponto de atenção, use o atalho para criar PDI.

Ao criar PDI:

- informe título;
- informe motivo;
- defina prazo, se necessário;
- vincule a avaliação, quando o PDI nasceu dela.

PDI sem avaliação também é permitido.

## 11. Como acompanhar PDI

Na aba `Desenvolvimento`, o sistema mostra:

- total de ações;
- ações abertas;
- ações concluídas;
- ações atrasadas;
- próxima ação;
- progresso do plano.

Para cada ação é possível:

- ver prazo;
- ver status;
- registrar observação de conclusão;
- marcar como concluída;
- reabrir, se necessário.

Item atrasado fica visível, mas não gera cobrança automática.

## 12. Como usar relatório

Acesse:

```text
RH -> Relatórios de avaliações
```

Use o relatório para localizar:

- avaliações em andamento;
- avaliações aguardando devolutiva;
- avaliações aguardando ciência;
- avaliações concluídas;
- avaliações com nota de atenção;
- avaliações com critério crítico;
- avaliações com PDI vinculado.

O relatório não é ranking. A média serve para acompanhamento, não para competição entre colaboradores.

## 13. O que não fazer

Não usar o módulo para:

- punição automática;
- advertência automática;
- ranking de colaboradores;
- comparação competitiva entre setores;
- promoção automática;
- desligamento automático;
- remuneração variável;
- avaliação 360.

## 14. Cuidados operacionais

- Escreva comentários objetivos e observáveis.
- Evite termos ofensivos ou subjetivos demais.
- Registre devolutiva somente depois da conversa.
- Use PDI para combinar ações práticas.
- Mantenha avaliação e PDI como apoio à gestão humana.
- Revise modelos periodicamente, sem criar excesso de critérios.
